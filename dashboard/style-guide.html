1. Design Philosophy

The Grid Iron Mind design system is built on three core principles:

Monochrome First

The entire interface uses a monochromatic palette of black, white, and grays. Color is introduced only when contextually meaningful:

  • Team-specific pages display team colors
  • Special call-to-action buttons use rainbow animation
  • Default state = no color

Glassmorphism & Depth

Components use semi-transparent backgrounds with backdrop blur to create a layered, modern aesthetic:

  • Cards: 80% opacity with blur(10px)
  • Inputs: 80% opacity with blur(10px)
  • Stat cards: 60% opacity with blur(10px)

Pixel-Perfect Buttons

Buttons use a unique skeuomorphic design with animated conic gradients. They're the visual centerpiece of interactions.

4. Buttons

Button Anatomy

Buttons use a 3-layer structure with animated conic gradients:

  1. Outer frame: White semi-transparent with gradient overlay
  2. Cap layer: Dark gray gradient with borders
  3. Inner text: Black rounded pill
<button class="btn-skeu">
    <span class="btn-cap">
        <span class="btn-text">Button Text</span>
    </span>
</button>

Button Variants

Variant Class When to Use
Default .btn-skeu General actions (no glow)
Rainbow .btn-skeu .btn-rainbow Primary CTAs (Add to Cart, Subscribe)

When to Use Team Colors

Do Use Team Colors

  • Team-specific actions (View Roster)

Don't Use Team Colors
  • Global navigation
  • Search/filter interfaces
  • Generic stat displays

10. Best Practices

General Guidelines

Do
  • Use monochrome by default
  • Apply backdrop-filter: blur(10px) to translucent elements
  • Use Outfit for all text except buttons
  • Keep shadows soft (0.04-0.08 opacity)
  • Add color only when contextually meaningful
  • Use CSS variables for maintainability
Don't
  • Add blue/green/red to generic UI
  • Use colored shadows except on colored buttons
  • Mix font families (no Comic Sans!)
  • Use hard shadows or heavy drop shadows
  • Apply team colors outside team context
  • Forget the 3-layer button structure

Accessibility

  • Ensure 4.5:1 contrast ratio for body text
  • Use 3:1 for large text (18px+)
  • Add focus indicators on all interactive elements
  • Don't rely on color alone to convey meaning

Performance

  • Use backdrop-filter sparingly (can impact performance)
  • Minimize animation complexity
  • Consider reducing blur radius on mobile
