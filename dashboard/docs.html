Best Practices

1. Always Use Pagination

Limit results to avoid overwhelming responses and improve performance.

✅ Good 
GET /api/v1/players?limit=50&offset=0

2. Filter Your Queries

Be specific with filters to get exactly what you need and reduce response size.

✅ Good 
GET /api/v1/players?position=QB&status=active&limit=20

3. Check Response Headers

Monitor cache status and rate limit headers for optimal usage.

Headers to Check 
X-Cache: HIT                    // Cached response (faster)
X-RateLimit-Remaining: 85       // Requests remaining
X-RateLimit-Reset: 1634567890   // Reset timestamp

4. Handle Errors Gracefully

Always implement proper error handling for network issues and API errors.

JavaScript 
async function fetchPlayers() {
  try {
    const response = await fetch('https://nfl.wearemachina.com/api/v1/players');

    if (!response.ok) {
      throw new Error(`HTTP ${response.status}: ${response.statusText}`);
    }

    const data = await response.json();
    return data.data;
  } catch (error) {
    console.error('Failed to fetch players:', error);
    return [];
  }
}

5. Cache Responses Locally

Implement client-side caching to reduce API calls and improve performance.

  • Cache team data for 1 hour (rarely changes)
  • Cache player data for 15 minutes
  • Cache game data for 5 minutes (during live games)
  • Don't cache AI predictions (always fresh)

6. Use Natural Language Queries Wisely

AI queries are powerful but use rate limits faster. Use them for complex questions.

✅ Good Use Cases for AI Queries

  • "Show me QBs with over 300 passing yards last week"
  • "Which teams have the most injuries right now?"
  • "Top 10 receivers by fantasy points this season"

7. Monitor Your Rate Limits

Track your usage to avoid hitting limits:

  • Standard endpoints: 100 requests/minute
  • AI endpoints: 10 requests/minute
  • Check X-RateLimit-Remaining header
  • Implement exponential backoff on 429 errors

What to Avoid

❌ Common Mistakes to Avoid

1. Don't Fetch Without Limits

❌ Bad 
GET /api/v1/players
// Returns all 2,500+ players - slow and wasteful
✅ Good 
GET /api/v1/players?limit=50&offset=0
// Returns 50 players - fast and efficient

2. Don't Ignore Error Responses

  • Assuming requests always succeed
  • Not checking HTTP status codes
  • Ignoring error messages in response body
  • Retrying failed requests without delays

3. Don't Abuse AI Endpoints

❌ Bad 
// Using AI query for simple lookups
POST /api/v1/garden/query
{"query": "get all teams"}

// This is wasteful - use regular endpoint instead
✅ Good 
// Use standard endpoint for simple queries
GET /api/v1/teams

// Reserve AI for complex questions
POST /api/v1/garden/query
{"query": "which QBs had the best performance in cold weather games?"}

4. Don't Hardcode API Keys

  • Never commit API keys to version control
  • Don't expose keys in client-side JavaScript
  • Don't share keys between environments
  • Use environment variables for key storage

5. Don't Poll Excessively

❌ Don't Do This

Polling every second during live games will quickly exhaust your rate limit and may result in temporary bans.

✅ Recommended Polling Intervals 
Live games:     Every 30-60 seconds
Pre-game:       Every 5-10 minutes
Off-season:     Every hour or less
Player stats:   Once per day

6. Don't Skip Pagination Metadata

Always check pagination info to know if more data exists:

Response Pagination 
{
  "data": [...],
  "meta": {
    "total": 2543,
    "limit": 50,
    "offset": 0,
    "timestamp": "2024-10-01T12:00:00Z"
  }
}

Example 2: Live Game Tracker

JavaScript 
class LiveGameTracker {
  constructor(season, week) {
    this.baseUrl = 'https://nfl.wearemachina.com/api/v1';
    this.season = season;
    this.week = week;
    this.pollInterval = 30000; // 30 seconds
  }

  async fetchGames() {
    const response = await fetch(
      `${this.baseUrl}/games?season=${this.season}&week=${this.week}`
    );
    const data = await response.json();
    return data.data;
  }

  async start(callback) {
    const poll = async () => {
      try {
        const games = await this.fetchGames();
        const liveGames = games.filter(g => g.status === 'in_progress');
        callback(liveGames);
      } catch (error) {
        console.error('Failed to fetch games:', error);
      }
    };

    // Initial fetch
    await poll();

    // Poll every 30 seconds
    this.intervalId = setInterval(poll, this.pollInterval);
  }

  stop() {
    if (this.intervalId) {
      clearInterval(this.intervalId);
    }
  }
}

// Usage
const tracker = new LiveGameTracker(2024, 5);
tracker.start((liveGames) => {
  console.log(`${liveGames.length} games in progress`);
  liveGames.forEach(game => {
    console.log(`${game.away_team} ${game.away_score} @ ${game.home_team} ${game.home_score}`);
  });
});

Example 3: Fantasy Football Dashboard

JavaScript 
async function buildFantasyDashboard(season, week) {
  const baseUrl = 'https://nfl.wearemachina.com/api/v1';

  // Fetch stat leaders
  const leadersResponse = await fetch(
    `${baseUrl}/stats/leaders?season=${season}&week=${week}`
  );
  const leaders = await leadersResponse.json();

  // Calculate fantasy points (PPR scoring)
  const calculateFantasyPoints = (stats) => {
    return (
      (stats.passing_yards * 0.04) +
      (stats.passing_tds * 4) +
      (stats.rushing_yards * 0.1) +
      (stats.rushing_tds * 6) +
      (stats.receptions * 1) + // PPR
      (stats.receiving_yards * 0.1) +
      (stats.receiving_tds * 6)
    );
  };

  // Process leaders by position
  const dashboard = {
    qb: leaders.data.passing_leaders.slice(0, 10).map(p => ({
      name: p.player_name,
      points: calculateFantasyPoints(p)
    })),
    rb: leaders.data.rushing_leaders.slice(0, 10).map(p => ({
      name: p.player_name,
      points: calculateFantasyPoints(p)
    })),
    wr: leaders.data.receiving_leaders.slice(0, 10).map(p => ({
      name: p.player_name,
      points: calculateFantasyPoints(p)
    }))
  };

  return dashboard;
}

// Usage
const dashboard = await buildFantasyDashboard(2024, 5);
console.log('Top QBs:', dashboard.qb);
console.log('Top RBs:', dashboard.rb);
console.log('Top WRs:', dashboard.wr);

Example 4: Weather Impact Analysis

Python 
import requests

def analyze_weather_impact(season, position='QB'):
    base_url = 'https://nfl.wearemachina.com/api/v1'

    # Get all games for the season
    games_response = requests.get(f'{base_url}/games?season={season}&status=final')
    games = games_response.json()['data']

    # Analyze performance by weather
    weather_stats = {
        'cold': {'games': 0, 'avg_yards': 0},
        'moderate': {'games': 0, 'avg_yards': 0},
        'hot': {'games': 0, 'avg_yards': 0}
    }

    for game in games:
        if game.get('weather'):
            temp = game['weather'].get('temperature_f', 70)

            # Categorize temperature
            category = 'moderate'
            if temp < 40:
                category = 'cold'
            elif temp > 75:
                category = 'hot'

            # Get game stats
            stats_response = requests.get(f'{base_url}/stats/game/{game["id"]}')
            stats = stats_response.json()['data']

            # Calculate average yards for position
            position_stats = [s for s in stats if s['position'] == position]
            if position_stats:
                avg_yards = sum(s.get('passing_yards', 0) for s in position_stats) / len(position_stats)
                weather_stats[category]['games'] += 1
                weather_stats[category]['avg_yards'] += avg_yards

    # Calculate averages
    for category in weather_stats:
        if weather_stats[category]['games'] > 0:
            games = weather_stats[category]['games']
            weather_stats[category]['avg_yards'] /= games

    return weather_stats

# Usage
impact = analyze_weather_impact(2024, 'QB')
print(f"Cold weather: {impact['cold']['avg_yards']:.1f} yards/game")
print(f"Moderate: {impact['moderate']['avg_yards']:.1f} yards/game")
print(f"Hot weather: {impact['hot']['avg_yards']:.1f} yards/game")

Handling Rate Limits

JavaScript 
async function fetchWithRateLimit(url) {
  const response = await fetch(url);

  // Check if rate limited
  if (response.status === 429) {
    const retryAfter = response.headers.get('Retry-After');
    console.log(`Rate limited. Retry after ${retryAfter} seconds`);

    // Wait and retry
    await new Promise(resolve => setTimeout(resolve, retryAfter * 1000));
    return fetchWithRateLimit(url);
  }

  // Check remaining requests
  const remaining = response.headers.get('X-RateLimit-Remaining');
  if (remaining < 10) {
    console.warn(`Only ${remaining} requests remaining`);
  }

  return response.json();
}

  • Implement exponential backoff on 429 errors

Comprehensive Error Handling

JavaScript 
async function apiRequest(url, options = {}) {
  try {
    const response = await fetch(url, options);

    // Check for HTTP errors
    if (!response.ok) {
      const error = await response.json();

      switch (response.status) {
        case 400:
          throw new Error(`Bad Request: ${error.error.message}`);
        case 401:
          throw new Error('Unauthorized: Invalid API key');
        case 404:
          throw new Error(`Not Found: ${error.error.message}`);
        case 429:
          const retryAfter = response.headers.get('Retry-After') || 60;
          throw new Error(`Rate limited. Retry after ${retryAfter}s`);
        case 500:
        case 503:
          throw new Error('Service temporarily unavailable');
        default:
          throw new Error(`API Error: ${error.error.message}`);
      }
    }

    return await response.json();
  } catch (error) {
    // Handle network errors
    if (error.name === 'TypeError') {
      console.error('Network error:', error.message);
      throw new Error('Network error. Please check your connection.');
    }

    // Re-throw API errors
    throw error;
  }
}

// Usage with retry logic
async function fetchWithRetry(url, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await apiRequest(url);
    } catch (error) {
      if (i === maxRetries - 1) throw error;

      // Exponential backoff
      const delay = Math.pow(2, i) * 1000;
      console.log(`Retry ${i + 1}/${maxRetries} after ${delay}ms`);
      await new Promise(resolve => setTimeout(resolve, delay));
    }
  }
}
