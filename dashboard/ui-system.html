Getting Started

Welcome to the Grid Iron Mind UI System. This comprehensive design system provides everything you need to build beautiful, consistent, and accessible user interfaces with a soft white glass aesthetic.

Installation via CDN
Include the UI System CSS in your HTML (easiest method):
<link rel="stylesheet" href="https://nfl.wearemachina.com/ui-system.css">
Installation via CSS Import
Or import it in your existing CSS file:
@import url('https://nfl.wearemachina.com/ui-system.css');
Download
Download the CSS file directly:
Download ui-system.css

Glassmorphism

Create beautiful frosted glass effects with our glassmorphism system.

Glass Morphism Effect

This card demonstrates the glassmorphism effect with backdrop blur, subtle transparency, and soft borders.

Glass Variables
/* Glassmorphism CSS Variables */
--glass-bg: rgba(255, 255, 255, 0.7);
--glass-bg-hover: rgba(255, 255, 255, 0.85);
--glass-border: rgba(255, 255, 255, 0.18);
--glass-shadow: 0 8px 32px rgba(0, 0, 0, 0.04);
--glass-shadow-hover: 0 12px 48px rgba(0, 0, 0, 0.08);

/* Apply glassmorphism */
.glass-element {
    background: var(--glass-bg);
    backdrop-filter: blur(20px) saturate(180%);
    -webkit-backdrop-filter: blur(20px) saturate(180%);
    border: 1px solid var(--glass-border);
    box-shadow: var(--glass-shadow);
}

Browser Support & Performance

Modern Browsers: Glassmorphism works best in Chrome, Safari, and Edge. Firefox has limited support.
Performance: Use sparingly on mobile devices as backdrop-filter can be GPU-intensive.
!
Fallback: Always provide a solid background color fallback for unsupported browsers.

Best Practices

Guidelines for building with this design system.

Do's

Use CSS Variables: Leverage the provided CSS custom properties for consistent theming.
Mobile First: Design for mobile screens first, then enhance for larger viewports.
Semantic HTML: Use proper HTML5 semantic elements for better accessibility and SEO.
Consistent Spacing: Stick to the spacing scale (xs, sm, md, lg, xl) for visual rhythm.

Don'ts

Avoid Hardcoded Colors: Don't use hex codes directly. Use CSS variables instead.
Don't Override Core Styles: Extend the system rather than fighting against it.
Avoid Excessive Glassmorphism: Too many glass elements can hurt performance.
Performance Tips
/* Optimize animations for 60fps */
.smooth-animation {
    will-change: transform;
    transform: translateZ(0);
}

/* Reduce backdrop-filter on mobile */
@media (max-width: 768px) {
    .glass-bg {
        backdrop-filter: blur(10px); /* Reduced from 20px */
    }
}

/* Use passive event listeners */
element.addEventListener('scroll', handler, { passive: true });

Accessibility

Building inclusive experiences for all users.

Accessibility Guidelines

Keyboard Navigation: All interactive elements are keyboard accessible with visible focus states.
Color Contrast: Text colors meet WCAG AA standards (4.5:1 for normal text).
Screen Readers: Use ARIA labels and semantic HTML for screen reader support.
Reduced Motion: Respects prefers-reduced-motion for users sensitive to animations.
Accessible Component Example
<!-- Accessible Button -->
<button
    class="btn-primary"
    aria-label="Save changes"
    aria-describedby="save-description">
    Save
</button>
<span id="save-description" class="sr-only">
    This will save your changes permanently
</span>

<!-- Accessible Form -->
<label for="email">Email Address</label>
<input
    type="email"
    id="email"
    aria-required="true"
    aria-invalid="false"
    aria-describedby="email-error">
<span id="email-error" role="alert"></span>
